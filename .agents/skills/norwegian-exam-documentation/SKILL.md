# Norwegian Exam Documentation

Use this skill when writing documentation, README files, user guides, exam scripts, or explanations for the examiner.

## Language

Write in Norwegian Bokmål unless the user asks for another language.

Target level:

- VG2 IT
- 17-18 year old student
- Professional but understandable
- Aim for grade 4-5

## Include where relevant

- User guide for employees
- IT admin guide
- Troubleshooting routine
- Backup routine
- Monitoring routine
- Security advice
- README
- Installation guide
- Exam explanation
- Reflection: advantages, disadvantages, improvements

## Tone

Use simple, clear and professional language.

Avoid:

- Overly academic language
- Fake-sounding explanations
- Too much advanced theory
- Unnecessary buzzwords

## Documentation structure

Prefer this structure:

1. Hva løsningen er
2. Hvorfor firmaet trenger den
3. Hvordan den fungerer
4. Hvordan den settes opp
5. Hvordan den testes
6. Hvordan brukere bruker den
7. Hvordan IT-avdelingen drifter den
8. Sikkerhet
9. Backup
10. Overvåking
11. Feilsøking
12. Forbedringer

## Exam explanation

Always include practical phrases the student can say to the examiner.

Example style:

"Jeg valgte denne løsningen fordi et firma trenger en ryddig måte å håndtere IT-problemer på. I stedet for at ansatte sender meldinger på e-post eller Teams, kan de lage en sak i systemet. Da får IT-avdelingen bedre oversikt og kan prioritere saker etter alvorlighet."

## Brukerstøtte focus

When writing user support documentation, include:

- Step-by-step instructions
- Simple screenshots placeholders if relevant
- Common problems
- What the user should do
- When the user should contact IT
