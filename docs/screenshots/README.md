# Screenshots for documentation

Legg screenshots her når UI-en er stabil. Dette hjelper mye for Brukerstøtte-delen av eksamen.

## Screenshots du bør ta

1. `employee-form.png` — ansatt-skjema før innsending.
2. `employee-success-state.png` — suksessmelding med ticket-ID.
3. `admin-dashboard.png` — admin dashboard med KPI-kort og tickets.
4. `ticket-detail-drawer.png` — åpnet ticket med detaljer og løsning.
5. `empty-state.png` — når filter gir ingen tickets.
6. `error-state.png` — når API/server er nede eller request feiler.
7. `mobile-form.png` — ansatt-skjema på smal skjerm hvis du rekker.

## Tips

- Ta screenshots etter at demo-data er seedet.
- Bruk samme vindusstørrelse for admin-bilder.
- Ikke vis ekte passord, tokens eller sensitiv informasjon.
- Bruk bildene i brukerveiledningen og demo-scriptet.
